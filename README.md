## health app

### Информационно-справочная система контроля параметров состояния здоровья

**health_app** — мобильное Android-приложение, разработанное в рамках курсовой работы по дисциплине «Разработка мобильных приложений».
Приложение предназначено для хранения и просмотра информации о состоянии здоровья пациентов с разделением ролей **врач / пациент**.

---

## Цель проекта

Разработка мобильного приложения, позволяющего:

* отслеживать состояние здоровья пациента;
* хранить медицинскую информацию в облачной базе данных;
* обеспечивать удобный и быстрый доступ к данным как для врача, так и для пациента.

---

## Основные возможности

### Для врача

* просмотр списка закреплённых пациентов;
* просмотр и редактирование диагнозов и истории болезни пациента;
* добавление пациентов по email;
* просмотр справочника заболеваний.

### Для пациента

* просмотр собственных диагнозов и истории болезни;
* просмотр списка врачей, за которыми он закреплён;
* просмотр справочника заболеваний.

---

## Ролевая модель доступа

| Роль    | Возможности                                                    |
| ------- | -------------------------------------------------------------- |
| Врач    | Просмотр пациентов, редактирование диагнозов и истории болезни |
| Пациент | Просмотр своих медицинских данных                              |

---

## Используемые технологии

* Язык программирования: Java
* Платформа: Android
* Минимальная версия Android: 7.0 (API 24)
* Среда разработки: Android Studio
* База данных: Firebase Firestore
* Аутентификация: Firebase Authentication

---

## Структура базы данных (Firebase)

### Коллекция `users`

Хранит данные пользователей:

* email
* role (doctor / patient)
* ФИО
* medicalHistory (анамнез)
* diseases (диагнозы)

### Коллекция `doctor_patient`

Хранит связи между врачами и пациентами:

* email врача
* email пациента

---

## Алгоритм работы приложения

1. Регистрация или авторизация пользователя.
2. Определение роли пользователя.
3. Отображение доступного функционала.
4. Просмотр и (при наличии прав) редактирование медицинских данных.
5. Работа с данными в реальном времени через Firebase.
